Files Structure

  • /ajax — Ajax files
  • /css — Template CSS
    • /demos — Demo-specific stylesheets
    • /examples — Example stylesheets
    • /fonts — Font faces, Font Awesome, etc.
    • /skins
      • default.css
      • … — Add your own custom skins here
  • /img — All images
  • /js — Template JavaScript
    • /demos — Scripts related to specific demos

  • /video — Video files

JavaScript

You can customize JavaScript settings by extending the plugin defaults — this makes updates easier since only one file needs to change.

Changing Default Settings

1) Change the "scroll to top" icon: js/custom.js

$.extend(theme.PluginScrollToTop.defaults, {
    iconClass: 'fa fa-chevron-up'
});

2) Change the "word rotate" default delay: js/custom.js

$.extend(theme.PluginWordRotate.defaults, {
    delay: 3000
});

All plugin and partial defaults can be found in js/theme.js:

Changing Plugin Options via Data Attributes

The template auto-initializes elements like sliders using a simple data-plugin-options attribute:

<div id="revolutionSlider" class="slider rev_slider" data-version="5.4.8"
     data-plugin-revolution-slider
     data-plugin-options="{'delay': 9000, 'gridwidth': 1170, 'gridheight': 670}">

This approach works the same way for most included plugins: Owl Carousel, Word Rotate, etc.

Disabling Auto-Initialization

Add the manual class or remove the data-plugin-* attribute to disable automatic initialization:

<div id="revolutionSlider" class="slider rev_slider manual" data-version="5.4.8">

Then initialize the plugin manually:

$(document).ready(function() {
    $('#revolutionSlider').revolution({
        delay: 9000,
        gridwidth: 1170,
        gridheight: 670
        // ...
    });
});

Removing a Plugin or Partial

To completely disable a plugin that has an initialize function, override it in js/custom.js:

theme.PluginScrollToTop.initialize = function() {};

HTML Structure

The template is built on Bootstrap 5 — getbootstrap.com/docs/

Bootstrap provides a responsive, mobile-first fluid grid system that scales up to 12 columns based on the device or viewport size.

<div class="row">
    <div class="col-md-12">
        Level 1 column
        <div class="row">
            <div class="col-md-6">Level 2</div>
            <div class="col-md-6">Level 2</div>
        </div>
    </div>
</div>

5-Column Grid Layout (Custom)

For a 5-column layout (not available natively in Bootstrap), use the custom col-md-1-5 class:

<div class="row">
    <div class="col-md-1-5">Column 1</div>
    <div class="col-md-1-5">Column 2</div>
    <div class="col-md-1-5">Column 3</div>
    <div class="col-md-1-5">Column 4</div>
    <div class="col-md-1-5">Column 5</div>
</div>

Customization & Styles

Colors

The Style Switcher supports unlimited skin colors using LESS (lesscss.org), which processes CSS dynamically in the demo. You don't need to use LESS in your own project — once you've customized your skin, copy the generated CSS and save it to /css/skins/your-skin-name.css.

→ Learn more: How to change the Skin

Layout Style (Wide / Boxed)

The default layout is Wide. To switch to the Boxed layout, add the boxed class to the <html> tag:

<html class="boxed">

You can also customize the background color or image in the boxed mode.

Color Scheme (Light / Dark)

The default color scheme is Light. To enable the Dark version, add the dark class to the <html> tag:

<html class="dark">

Fonts

To change the website font, locate the Google Fonts link in the <head> of your HTML file:

<link href="http://fonts.googleapis.com/css?family=Open+Sans:300,400,600,700,800|Shadows+Into+Light"
      rel="stylesheet" type="text/css">

To change the font, visit fonts.google.com, choose a font, and replace the link above with the one provided.

Skin

You can generate a custom skin color using the Style Switcher in the template preview:

Copy the generated CSS and save it as a skin file at css/skins/default.css.

Note 1: If you are using a niche demo (Construction, Medical, Law Firm, Architecture, etc.) as your starting point, update the corresponding niche skin file instead of default.css. For example: Construction → css/skins/skin-construction.css, Medical → css/skins/skin-medical.css.
Note 2: Make sure to generate the skin using the correct version URL of the preview. For example: https://www.okler.net/previews/porto/9.9.1/

Video Background

Video backgrounds are powered by the Vide plugin — github.com/VodkaBears/Vide.

Here's a usage example:

<section class="video section section-video"
         data-video-path="video/memory-of-a-woman"
         data-plugin-video-background
         data-plugin-options="{'posterType': 'jpg', 'position': '50% 50%', 'overlay': true}">

Prepare your video in multiple formats (.webm, .mp4) for cross-browser compatibility. Also provide a poster image in .jpg, .png, or .gif format and reference it via the data-video-path attribute.

Effects

You can add scroll-triggered animations to any element. When the element enters the viewport, the animation plays automatically.

Usage:

<div class="appear-animation"
     data-appear-animation="bounceIn"
     data-appear-animation-delay="200">
    …
</div>

The data-appear-animation-delay attribute is optional and is set in milliseconds.

Any animation name from Animate.css can be used as the value for data-appear-animation.

Facebook Comments

Facebook Comments require the page to be served from a local or online server (e.g., Apache). They will not work when opening HTML files directly in a browser.

A working example can be found on the page blog-post-comments-facebook.html, which shows the Facebook developer page comments by default.

To display comments for your own page, update the data-href attribute to your page URL:

data-href="https://www.yourdomain.com/your-page/"

Adjust the number of visible comments using data-numposts:

data-numposts="7"

The default language is English (en_US). To change it, modify the language code in the Facebook script tag placed right after the <body> tag. For Spanish, for example:

To add Facebook Comments to another page, copy the JavaScript block above and paste it after the opening <body> tag. Then add the following HTML where you want the comments to appear:

<h4 class="mb-3">
    Comments (<fb:comments-count href="https://www.yourdomain.com/your-page/"></fb:comments-count>)
</h4>
<div class="fb-comments w-100"
     data-href="https://www.yourdomain.com/your-page/"
     data-numposts="7" data-width="100%"></div>

Disqus Comments

Step 1 — Create an account and forum at disqus.com.

Step 2 — Add the Disqus scripts before the closing </body> tag, replacing YOUR_DISQUS_FORUM_SHORTNAME with your forum's short name:

<!-- Disqus Scripts -->
<script>
    var disqus_subdomain = 'YOUR_DISQUS_FORUM_SHORTNAME';

    (function() { // DON'T EDIT BELOW THIS LINE
    var d = document, s = d.createElement('script');
    s.src = 'https://' + disqus_subdomain + '.disqus.com/embed.js';
    s.setAttribute('data-timestamp', +new Date());
    (d.head || d.body).appendChild(s);
    })();
</script>
<script id="dsq-count-scr"
        src="//YOUR_DISQUS_FORUM_SHORTNAME.disqus.com/count.js" async></script>

Step 3 — Add the Disqus thread container where you want comments to appear:

<div id="disqus_thread"></div>
